## 결과 (2026-04-14)

실험 결과 공유 후 팀 논의 결과:

1. **모델:** gpt-4o-mini **유지** 결정
2. **허용 오차:** 기본값 1% → **10%로 확대**

### 결정 사유

**핵심 원칙:** 비용 효율성을 최우선으로 고려

- 본 실험에서 gpt-4o의 품질 우위 확인
- 그러나 **비용 대비 효율** 관점에서 mini가 더 합리적
  - mini 비용: gpt-4o의 약 5% (95% 절감)
  - 품질 차이는 오차 범위 조정으로 보완 가능
- 프로덕트 특성상 **빈번한 호출 예상**되어 장기 운영 시 비용 차이 누적
- 제약 조건 하에서의 최적 설계 = mini + 오차 확대 조합

### 제약 조건 안에서의 최선: 오차 확대로 보완

mini의 낮은 정확도를 **오차 범위 확대**로 상쇄:

| 목표 | mini 결과들 | 기존 범위 (1%) | 새 범위 (10%) | 기존 성공 | 새 성공 |
|---|---|---|---|---|---|
| 500자 | 477/462/463 | 495~500 | 450~500 | 0/3 | **3/3 ✅** |
| 800자 | 707/745/772 | 792~800 | 720~800 | 0/3 | **2/3 ✅** |
| 1000자 | 919/957/997 | 990~1000 | 900~1000 | 1/3 | **3/3 ✅** |
| 1500자 | 1143/1471/1367 | 1485~1500 | 1350~1500 | 0/3 | **2/3 ✅** |
| **전체** | | | | **1/12 (8%)** | **10/12 (83%)** |

**설정 변경만으로 성공률 8% → 83% 달성 (10배 향상).**

재측정 없이 기존 데이터만으로도 효과 검증됨.

## 교훈

### 엔지니어링 관점

1. **가장 좋은 기술이 항상 정답은 아님** — 제약 조건이 기술 선택을 결정
2. **수치로 결정하기** — 추측이 아닌 측정 데이터로 "mini + 오차 10% = 83%" 근거 확보
3. **유연한 설계의 가치** — 코드 구조가 모듈화되어 있어 모델/버퍼 변경이 한 줄로 가능

### 프로젝트 관점

- **비용 효율성** 관점에서 초기부터 저렴한 모델을 검증하는 것이 합리적
- 품질 요구사항을 **범위 허용도로 완화**하는 것도 유효한 설계 전략
- 실험 데이터가 팀 의사결정의 **근거**가 됨 (본 실험 결과가 팀 결정에 직접 활용됨)
